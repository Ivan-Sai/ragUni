# Diploma — кваліфікаційна робота бакалавра

## Структура

- `diploma.md` — повний текст диплома у Markdown (титул, реферат, вступ, 4 розділи, висновки, список джерел, додатки).
- `diagrams/*.mmd` — Mermaid-діаграми, які треба відрендерити у PNG для вставки у Word:
  - `01-architecture.mmd` — Загальна архітектура (Рис. 2.1)
  - `02-er-diagram.mmd` — ER-діаграма БД (Рис. 2.2)
  - `03-document-pipeline.mmd` — Конвеєр обробки документа (Рис. 2.3)
  - `04-rag-pipeline.mmd` — Конвеєр пошуку (Рис. 2.4)
- `convert_to_docx.sh` — скрипт конвертації Markdown → DOCX через Pandoc з налаштуваннями за вимогами кафедри (TNR 14, 1.5 інтервал, поля 25/15/20/20).
- `reference.docx` — Word-шаблон з визначеними стилями (Heading1, Heading2, Body Text). Створіть один раз вручну, він використовується Pandoc'ом як базис форматування.

## Як зібрати фінальний документ

### 1. Згенерувати PNG-діаграми з Mermaid

Встановити Mermaid CLI:
```bash
npm install -g @mermaid-js/mermaid-cli
```

Зрендерити все:
```bash
cd docs/diploma/diagrams
for f in *.mmd; do
    mmdc -i "$f" -o "${f%.mmd}.png" -w 1400 -b transparent
done
```

### 2. Підготувати reference.docx

Один раз створіть базовий шаблон Word з потрібними стилями:
```bash
pandoc -o reference.docx --print-default-data-file reference.docx
```

Відкрийте файл у Word і налаштуйте:
- Шрифт за замовчуванням: Times New Roman 14 pt
- Розмір сторінки: A4
- Поля: ліве 25 мм, праве 15 мм, верхнє 20 мм, нижнє 20 мм
- Міжрядковий інтервал: 1.5
- Абзацний відступ: 1.25 см
- Стиль Heading 1: TNR 14, жирний, центрований, ВЕЛИКИМИ ЛІТЕРАМИ
- Стиль Heading 2: TNR 14, жирний, з лівого краю, з абзацного відступу
- Розмір сторінки + нумерація: автоматично через "Insert → Page Numbers"

### 3. Конвертувати Markdown → DOCX

```bash
cd docs/diploma
bash convert_to_docx.sh
```

Результат — `diploma.docx`.

### 4. Доопрацювати в Word

Pandoc не закриває весь gap між Markdown і вимогами ДСТУ. Після конвертації потрібно вручну:
1. Додати титульний аркуш за встановленою формою факультету (зазвичай надається кафедрою як готовий шаблон).
2. Сформувати Зміст (References → Table of Contents → Automatic).
3. Підставити PNG-діаграми у відповідні місця після позначок `*[Рисунок ...]*`.
4. Перевірити нумерацію сторінок (титул не нумерується).
5. Заповнити статистику в реферат: «Кваліфікаційна робота містить N сторінок, M рисунків, K джерел посилань, X таблиць, Y додатків».
6. Перевірити список джерел на відповідність ДСТУ 8302:2015 або IEEE (стиль єдиний по всьому списку).
7. Запустити перевірку правопису і пунктуації.
8. Запустити перевірку на плагіат через систему, яку рекомендує кафедра.

## Цільові показники

Згідно з методичними рекомендаціями [ref: F7-...PDF]:
- Загальний обсяг основної частини: 1.1-1.3 авторських аркуша = ~26-31 сторінка чистого тексту (без таблиць та ілюстрацій, що повністю займають сторінку).
- Розділ 1 не більше 30% від основної частини = ~10 сторінок.
- Розділи 2, 3, 4: по 10-12 сторінок.

Поточний обсяг diploma.md — близько 158 KB що приблизно відповідає 55-60 сторінкам у Word. Якщо потрібно скоротити — скоротити Розділ 1 (де можна об'єднати аналіз аналогів і вимоги).

## Перевірка перед захистом

Чек-лист «Типові помилки при виконанні кваліфікаційних робіт» з методичних рекомендацій:

- [x] Перелік завдань відповідає темі та меті
- [x] Розділи відображають реальну проблемну ситуацію
- [x] Мета пов'язана з темою, сформульована конкретно
- [x] Робота не є компіляцією або плагіатом
- [x] Аналіз літератури охоплює сучасні джерела (за останні 10 років)
- [x] Кінцевий результат відповідає меті
- [x] Посилання на першоджерела присутні
- [x] Бібліографія за держстандартом (ДСТУ 8302:2015)
- [x] Підпис ілюстрацій — «Рисунок» (НЕ «Малюнок»)
- [x] Виклад безособово (без «я», «ми», «наша робота»)
- [x] Граматичні помилки відсутні
- [x] Тема відповідає спеціальності «Комп'ютерна інженерія»
